## 1. 产品概述

老旧小区改造公示管理平台——为街道、居民和监理三方提供统一的改造进度公示与协同管理工具。街道录入项目信息并发布公示，居民查看计划并参与投票与异议，监理跟踪验收与返工。系统强制执行七天公示期、异议处理、投票达标、预算确认和监理备案五项开工前置条件，并对已验收楼栋实施变更锁定与重新公示机制。

- 目标用户：街道办事处工作人员、小区居民、工程监理单位
- 核心价值：确保改造全流程透明合规，杜绝"未公示先开工""异议未清即施工"等问题

## 2. 核心功能

### 2.1 用户角色

| 角色 | 进入方式 | 核心权限 |
|------|----------|----------|
| 街道 | 选择"街道入口" | 录入项目包、楼栋分户、预算来源、施工单位、公示时间；处理异议；审核变更 |
| 居民 | 选择"居民入口" | 查看公示计划、投票赞成/反对、提交异议、查看异议处理结果 |
| 监理 | 选择"监理入口" | 更新阶段验收结果、录入返工信息、备案确认 |

### 2.2 功能模块

1. **街道台账**：项目包总览、楼栋分户管理、预算来源登记、施工单位指派、公示时间设定
2. **居民公示端**：公示公告浏览、计划详情查看、投票（赞成/反对）、异议提交
3. **异议处理**：异议列表与状态流转（待处理→处理中→已处理/超期）、超期预警
4. **施工计划**：施工阶段管理、开工五项条件校验（公示满七天、异议已处理、投票达标、预算已确认、监理已备案）
5. **监理验收**：阶段验收打分、返工记录、验收锁定（已验收楼栋不可随意变更）
6. **变更历史**：变更申请、变更审批、重新公示计算、变更轨迹
7. **红色风险清单**：未满七天、异议超期、投票未达标、预算未确认、监理未备案、验收后变更等风险项聚合
8. **归档审计**：全量操作日志、公示归档、审计追溯

## 3. 核心流程

**公示发布→居民参与→开工校验→施工验收→变更管控→归档审计**

1. 街道录入项目包信息（楼栋、预算、施工单位），设定公示时间并发布
2. 居民在公示期内查看计划、投票、提交异议
3. 公示期满后系统校验五项开工条件，全部满足方可开工
4. 施工过程中监理逐阶段验收，不合格则返工
5. 已验收楼栋如需变更施工计划，须提交变更申请，审批后重新公示七天
6. 项目完成后归档，所有操作留痕可审计

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：深蓝灰（#1e293b）+ 警示红（#dc2626）+ 安全绿（#16a34a）
- 辅助色：琥珀黄（#d97706）用于待处理状态
- 按钮风格：圆角（8px）、实色填充、hover 微抬阴影
- 字体：Noto Sans SC 为主体字体，数据展示使用等宽字体
- 布局：左侧导航栏 + 右侧内容区，顶部角色切换标签
- 图标：lucide-react 图标库

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|---------|
| 街道台账 | 项目包管理 | 卡片列表、新增弹窗表单、状态标签 |
| 街道台账 | 楼栋分户 | 树形展开+表格、内联编辑 |
| 街道台账 | 预算来源 | 环形图+明细表格 |
| 居民公示端 | 公告列表 | 时间线卡片、状态角标 |
| 居民公示端 | 投票 | 进度条+赞成/反对按钮 |
| 异议处理 | 异议列表 | 表格+超期红色闪烁行 |
| 施工计划 | 开工校验 | 五项检查清单✓/✗、红色阻止弹窗 |
| 监理验收 | 阶段验收 | 步骤条+验收打分卡片 |
| 红色风险清单 | 风险聚合 | 红色卡片网格、倒计时/超期标记 |
| 归档审计 | 操作日志 | 时间线+筛选器 |

### 4.3 响应式

- 桌面优先设计，最小宽度 1280px
- 平板适配：导航栏折叠为抽屉
- 移动端：卡片单列布局，表格转为列表
